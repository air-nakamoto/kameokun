# Claude Project からのドキュメント取り込み手順

このプロジェクトでは、これまで Claude Project に入れていた亀奥問題・亀尾くん問題用のドキュメントを、GitHub 上で再利用できる形に整理していきます。

## できること

貼り付けられたドキュメントに対して、次の処理を行います。

- 内容を読み取り、目的・ルール・問題素材・口調・禁止事項などに分類する
- 重複している説明や古い指示を整理する
- GitHub で管理しやすい Markdown ファイルに分割する
- ボットに渡すためのシステムプロンプト、運用ルール、問題テンプレートに変換する
- 未確定の内容や追加確認が必要な内容を TODO として残す
- 変更内容を Git のコミットと Pull Request として記録する

## 貼り付けてもらうときの形式

ドキュメントは一度に全部貼り付けても、複数回に分けても構いません。複数回に分ける場合は、先頭に通し番号を付けると整理しやすくなります。

例:

```text
[Claude Project docs 1/4]
ここにドキュメント本文
```

```text
[Claude Project docs 2/4]
ここに続き
```

タイトルや用途が分かる場合は、次のように見出しを付けてください。

```text
# 亀奥問題の基本ルール
...

# ボットの口調
...

# 出題例
...
```

## 取り込み時の分類先

貼り付けられた内容は、必要に応じて次の場所へ整理します。

| 分類 | 保存先 | 内容 |
| --- | --- | --- |
| プロジェクト全体の説明 | `README.md` | 目的、全体像、運用の入口 |
| 移行・運用手順 | `docs/` | GitHub 移行、Claude Project からの取り込み、レビュー手順 |
| ボット向け指示 | `bot/` | システムプロンプト、応答ルール、禁止事項 |
| 問題テンプレート | `problems/template.md` | 問題本文、答え、解説、メタ情報の型 |
| 個別の問題 | `problems/` | 問題ごとの Markdown ファイル |
| 未整理メモ | `docs/import-notes.md` | 判断保留、後で確認する内容 |

`bot/` や `docs/import-notes.md` は、実際に必要になった時点で作成します。


## 複数回貼り付けるときの重複処理

Claude Project からの移行では、同じルール、似た説明、過去版の問題文が複数回出てくる可能性があります。その場合は、次の方針で処理します。

1. まず貼り付けられた原文を、受け取った順番が分かる形で確認する
2. 既存ファイルに同じ内容があるかを確認する
3. 完全に同じ内容は重複として統合する
4. 似ているが表現や条件が違う内容は、勝手に片方を削除せず差分を残す
5. 新旧関係が分かる場合は、新しい内容を本文に反映し、古い内容は移行メモに残す
6. どちらを採用すべきか判断できない場合は、`docs/import-notes.md` に確認事項として残す

貼り付ける側で重複を取り除く必要はありません。重複や揺れを含んだまま貼り付けてもらい、こちらで分類・統合・保留に分けます。

## 処理方針

1. 原文の意味を勝手に変えない
2. 判断に迷う内容は削除せず、保留メモとして残す
3. ボットの挙動に関わる指示は、通常の説明文と分けて管理する
4. 問題本文・答え・解説は、個別ファイルとして追跡できる形にする
5. GitHub 上でレビューできるよう、変更理由を Pull Request に書く

## 次に貼り付けてもらいたいもの

最初は、Claude Project に入れていたドキュメントのうち、次のどれかから貼り付けてください。

- ボットに最初に読ませていた基本指示
- 亀奥問題・亀尾くん問題のルール説明
- 代表的な出題例と回答例
- 絶対に守らせたい禁止事項や注意事項

貼り付け後、こちらで内容を分類し、必要なファイルへ反映します。
