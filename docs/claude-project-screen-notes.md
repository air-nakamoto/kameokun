# Claude Project 画面メモ

このメモは、共有された Claude Project 画面から読み取れる移行対象の構造を整理したものです。

## 画面全体

- 左側には Claude のプロジェクト一覧が表示されている。
- 右側には選択中プロジェクトの説明、手順、ファイル一覧が表示されている。
- 選択中のプロジェクト名は「オリジナル亀夫君問題を出題します」と読み取れる。
- 中央の入力欄から、このプロジェクトに対してチャットを開始できる。

## 左側のプロジェクト一覧

左側には複数の関連プロジェクトが並んでいる。確認できる名前には次のようなものがある。

- Cute Sea Turtle Wallpaper Design
- Ineffective Work Communication
- Kame-kun Problem Game
- Kame-o Mystery Game
- 亀夫君問題 New Challenge
- Kame-kun Problem Game Design
- Communication Report Challenge

「Kame-kun」「Kame-o」「亀夫君問題」など近い名前のプロジェクトが複数あるため、移行時にはどのプロジェクトが最新版・本命なのかを確認する必要がある。

## 右側の手順欄

右側上部の「手順」には、プロジェクトの基本指示らしい長いテキストが登録されている。画面上では一部だけが見えており、次のような内容が読み取れる。

- 新作オリジナル亀夫君問題出題専用ボット
- 統合ガイドライン
- 水平思考ユニット
- うみがめ部版
- うみがめ部のコンセプト
- 理念

これは、GitHub 側ではボット向けのシステムプロンプトまたは運用ルールとして `bot/` 配下に分離して管理する候補になる。

## 右側のファイル欄

右側下部の「ファイル」には、プロジェクトに読み込ませている資料がカード形式で並んでいる。画面上では、プロジェクト容量の約 3% を使用中で、インデックス作成中と表示されている。

確認できるファイルには、次のようなものがある。

- 新作オリジナル亀夫君問題出題専用ボット 統合ガイドライン系の TXT
- 亀夫君問題出題テンプレート集.txt
- 中本アイアールロ調ガイド
- 亀夫君問題サンプル5「プロポーズ大失敗」.txt
- プロジェクトの重圧＿ブレイク.txt
- 亀夫君問題サンプル2「過去を忘れて」.txt
- 亀夫君問題サンプル3「大切に使うって約束してね」.txt
- 教えて亀夫君！【ナカメタケの水平思考Radio】系の TXT
- 亀夫君問題サンプル4「伝わらない報告」.txt

## GitHub 移行での扱い

この画面から見る限り、移行対象は大きく次の 3 種類に分けられる。

1. プロジェクトの手順欄にあるボット全体の指示
2. ファイル欄にあるガイドライン・テンプレート・文体ガイド
3. ファイル欄にある個別サンプル問題

移行時は、手順欄の内容とファイル欄の内容を混ぜずに扱う。特に、ボットの振る舞いを決める指示、問題作成のテンプレート、個別サンプル問題は別ファイルとして管理する。

## 次に貼り付けてほしいもの

まずは右側の「手順」欄に入っている全文を貼り付けるのがよい。これはボット全体の挙動を決める中心情報だと考えられるため、最初に GitHub 側の基本方針へ変換しやすい。

その次に、ファイル欄のガイドラインやテンプレート、最後にサンプル問題を順番に貼り付けると整理しやすい。
